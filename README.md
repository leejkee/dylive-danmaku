## 抖音直播弹幕抓取扩展 (dylive-danmaku)

这是一个用于实时抓取和提取抖音直播间弹幕数据的 **Chrome** 浏览器扩展。本项目通过向页面注入探测脚本，直接拦截底层的 **React Fiber** 节点数据，能够有效绕过前端 CSS 类名混淆以及虚拟列表（**Virtual List**）带来的 **DOM** 节点复用限制，实现高稳定性的全量明文数据提取。

### 功能特点

- **多类型消息捕获**：支持抓取普通聊天弹幕、特权屏幕弹幕、礼物赠送、用户入场、直播间点赞、关注主播、粉丝团升级以及福袋等多种事件。
- **突破虚拟列表限制**：不依赖传统的 **DOM** 节点增删监听（**MutationObserver** `addedNodes`），而是通过定时轮询现存 DOM 并深度遍历 **React Fiber** 树，完美解决弹幕快速滚动时的漏抓问题。
- **纯前端拦截**：无需配置复杂的抓包工具或证书，直接在浏览器运行时环境中提取明文数据。
- **内存安全**：内置严格的消息 ID 去重机制和 **Set** 缓存容量控制，防止长时间挂机导致浏览器内存泄漏崩溃。
- **独立控制面板**：提供直观的 **Popup** 弹窗界面，可随时开启或暂停抓取，并实时预览抓取到的结构化数据。

### 安装与使用 (普通用户)

由于浏览器的安全策略限制，本项目通过源码压缩包的方式进行分发。

1. 前往本仓库的 **Releases** 页面。
2. 下载最新版本的 `dylive-danmaku-vX.X.X.zip` 压缩包文件。
3. 将下载的压缩包解压到你电脑上的一个固定文件夹中（请不要删除该文件夹）。
4. 打开 **Chrome** 浏览器（或 **Edge** 等基于 **Chromium** 的浏览器），在地址栏输入 `chrome://extensions/` 并回车。
5. 在页面右上角，开启 **开发者模式** 开关。
6. 点击左上角的 **加载已解压的扩展程序** 按钮。
7. 选择你刚刚解压的文件夹，即可完成安装。

### 使用说明

安装完成后，在浏览器打开任意抖音直播间网页。点击浏览器右上角扩展栏中的本扩展图标，在弹出的面板中点击**开启抓取**即可看到实时数据流。

### 开发指南

本项目使用 **TypeScript** 编写，并使用 **esbuild** 进行极速构建，打包输出符合 **Chrome Extension Manifest V3** 标准的代码。

#### 环境要求

- **Node.js**（推荐 v20 或以上版本）
- **pnpm**（包管理器）

#### 快速开始

1. 克隆本仓库到本地：

   ```bash
   git clone <你的仓库地址>
   cd dylive-danmaku
   ```

2. 安装依赖项：

   ```bash
   pnpm install
   ```

3. 编译构建：

   ```bash
   pnpm run build
   ```

   执行完成后，所有编译好的扩展文件将生成在根目录的 `dist` 文件夹下。在 Chrome 扩展管理页面加载该 `dist` 文件夹即可进行本地调试。

4. 打包发布：

   ```bash
   pnpm run pack
   ```

   该命令会先执行构建，随后使用 **archiver** 将构建产物打包成 `.zip` 文件，存放在 `release` 目录中。

#### 目录结构说明

- `src/background/`：包含 **Service Worker** 脚本，负责在后台持续接收并暂存弹幕数据，保证控制面板关闭时数据不丢失。
- `src/content-scripts/`：包含内容脚本，负责在网页加载时注入探测器，并在探测器和后台服务之间建立通信桥梁。
- `src/inject-scripts/`：包含核心注入脚本，负责在网页上下文中运行，查找并遍历 **React Fiber** 树提取真实数据。
- `src/popup/`：包含控制面板的交互逻辑脚本。
- `public/`：存放静态资源，如 `manifest.json`、**HTML** 页面、**CSS** 样式表和图标等。
- `scripts/`：存放用于构建和打包的 **Node.js** 工具脚本。
- `.github/workflows/`：存放 **GitHub Actions** 自动化 CI/CD 发布脚本。
